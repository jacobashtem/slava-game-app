## ZADANIE 1: Rozdziel trigger ON_DAMAGE w constants.ts

Obecny `EffectTrigger.ON_DAMAGE` musi zostać rozdzielony na DWA osobne triggery:

```typescript
// USUŃ:
ON_DAMAGE = 'on_damage',

// DODAJ:
ON_DAMAGE_DEALT = 'on_damage_dealt',       // Gdy TA istota ZADAJE obrażenia (Biali Ludzie, Bezkost, Homen, Strzyga, Bazyliszek)
ON_DAMAGE_RECEIVED = 'on_damage_received',  // Gdy TA istota OTRZYMUJE obrażenia (Bugaj, Brzegina)
```

Po zmianie zaktualizuj wszystkie odwołania w EffectRegistry:
- `bugaj_def_to_atk`: zmień trigger na `ON_DAMAGE_RECEIVED`
- `brzegina_shield_for_gold`: zmień trigger na `ON_DAMAGE_RECEIVED` (bo to sojusznik otrzymuje obrażenia)
- `biali_ludzie_wound_disarm`: zmień trigger na `ON_DAMAGE_DEALT`
- Wszystkie inne efekty używające `ON_DAMAGE`: przejrzyj i przydziel poprawny wariant

## ZADANIE 3: Dodaj dodatkowe reactionTriggery

W nowym pliku JSON karty mają pole `reactionTrigger` wewnątrz `abilities[]`.
Oto pełna lista reactionTriggerów które muszą być obsługiwane przez silnik:

```typescript
// Dodaj do EffectTrigger enum (jeśli brakuje):
ON_ALLY_ATTACKED = 'on_ally_attacked',     // Chowaniec (#10), Niedźwiedziołak (#77), Wielkolud (#49)
ON_ALLY_DAMAGED = 'on_ally_damaged',        // Brzegina (#6)
ON_ALLY_SURVIVES = 'on_ally_survives',      // Znachor (#52) — sojusznik przeżyje atak
ON_ALLY_SPELLED = 'on_ally_spelled',        // Bzionek (#64) — zaklęcie celuje w sojusznika
ON_SPELL_CAST = 'on_spell_cast',            // Czarownica (#36) — ktokolwiek rzuca zaklęcie
ON_GOLD_SPENT = 'on_gold_spent',            // Najemnik (#45) — gracz wydaje PS
ON_DEFENSE_ZERO = 'on_defense_zero',        // Czart (#99) — obrona spadnie do 0
ON_COMBAT = 'on_combat',                    // Wisielec (#88) — byle kontakt bojowy
```

Nie wszystkie muszą być zaimplementowane od razu. Ale enum musi je zawierać,
żeby EffectRegistry mógł się do nich odwoływać.

## ZADANIE 7: Obsłuż pole abilities[] w UI

Pole `abilities[]` w JSON jest przeznaczone DO WYŚWIETLENIA na karcie w UI.
Silnik czyta `effectId` → EffectRegistry. UI czyta `abilities[]` → wyświetla tekst.

W komponencie `CreatureCard.vue`:
- Wyświetl abilities jako listę pod statystykami
- Trigger jako tag/badge: WEJŚCIE, AKCJA, AURA, REAKCJA, ŚMIERĆ
- Mapowanie triggerów JSON → UI label:
  - ON_PLAY → "WEJŚCIE"
  - ACTION → "AKCJA"  
  - AURA → "AURA"
  - REACTION → "REAKCJA"
  - ON_DEATH → "ŚMIERĆ"
  - ON_KILL → "ZABÓJSTWO"
  - ON_TURN_START → "START TURY"
  - ON_TURN_END → "KONIEC TURY"
  - ON_ANY_DEATH → "KAŻDA ŚMIERĆ"
  - ON_ATTACK → "ATAK"
  - ON_ENEMY_PLAY → "ZASADZKA"
  - ENEMY_ACTION → "AKCJA WROGA"
